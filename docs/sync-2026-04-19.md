## Change 1 — `src/tools/email.py`: Inbox reads by date window

**Problem:** Reading only `UNSEEN` messages meant any email opened in Proton Mail before the CRM processed it was permanently invisible to the agent.

**Fix:** Search by `SINCE <date>` (last 14 days) instead. Deduplication is handled by the `inbox_messages` table on insert, so re-fetching is safe.

**Also:** Extract HTML body as fallback when no `text/plain` part exists.

```python
def read_inbox(limit: int = 50, since_days: int = 14) -> list[dict]:
    from datetime import timedelta
    since_date = (datetime.now(timezone.utc) - timedelta(days=since_days)).strftime("%d-%b-%Y")
    # ...
    _, data = imap.search(None, f"SINCE {since_date}")  # was: "UNSEEN"
```

HTML fallback (add after the `is_multipart()` block):

```python
html_body = ""
if parsed.is_multipart():
    for part in parsed.walk():
        ct = part.get_content_type()
        if ct == "text/plain" and not body:
            body = part.get_payload(decode=True).decode("utf-8", errors="replace")
        elif ct == "text/html" and not html_body:
            html_body = part.get_payload(decode=True).decode("utf-8", errors="replace")
else:
    body = parsed.get_payload(decode=True).decode("utf-8", errors="replace")

if not body.strip() and html_body:
    import html as html_mod
    import re
    text = re.sub(r"<[^>]+>", " ", html_body)
    body = re.sub(r" {2,}", " ", html_mod.unescape(text)).strip()
```

## Change 2 — `src/supervisor/run_followup.py`: Inbox backlog merge

**Problem:** Messages that were fetched from IMAP but not processed (e.g. agent crashed, skipped due to limit) sat in `inbox_messages` unprocessed forever.

**Fix:** After the fresh IMAP fetch, query `inbox_messages WHERE processed = false` and merge any IDs not already in the fresh batch.

Requires a `get_unprocessed_inbox` tool function (see your `src/tools/` — add it if missing):

```python
def get_unprocessed_inbox() -> list[dict]:
    with db() as conn:
        cur = conn.cursor()
        cur.execute("""
            SELECT id, message_id, from_email, subject, body, received_at
            FROM inbox_messages
            WHERE processed = false
            ORDER BY received_at ASC
        """)
        return [dict(r) for r in cur.fetchall()]
```

Then in `run_followup.py`, replace the `fetch_inbox=read_inbox` line with a wrapper:

```python
def fetch_inbox_with_backlog(limit: int = 50) -> list[dict]:
    new_messages = read_inbox(limit=limit)
    new_ids = {m["id"] for m in new_messages}
    backlog = [m for m in get_unprocessed_inbox() if m["id"] not in new_ids]
    if backlog:
        logger.info("fetch_inbox_with_backlog: %d backlog message(s) added", len(backlog))
    return new_messages + backlog

agent = create_followup_agent(
    llm=get_llm("claude"),
    fetch_inbox=fetch_inbox_with_backlog,
    # ...
)
```

### 4b — Add `_fetch_rejected` function

```python
def _fetch_rejected(conn) -> list[dict]:
    cur = conn.cursor()
    cur.execute("""
        SELECT
            aq.id, aq.draft_subject, aq.created_at, aq.reviewed_at, aq.reviewer_note,
            c.id AS contact_id, c.name AS contact_name, c.city, c.email
        FROM approval_queue aq
        JOIN contacts c ON c.id = aq.contact_id
        WHERE aq.status = 'rejected'
        ORDER BY aq.reviewed_at DESC
    """)
    return [dict(row) for row in cur.fetchall()]
```

## Change 6 — `src/tools/memory.py`: Fix Open Brain auth header

The Python MCP client requires both headers. Change:
